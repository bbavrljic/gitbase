Git config cascade

  • On Win there are actually 4!
    • :ProgramData (win only)
    • system:Installation Folder (win only)
    • global:Home (all)
    • local:Repo (all)

                # to list config source, use:

                $git config --show-origin --list

Defining shortcuts

Defining shortcuts

  • edit file ~/.bashrc
    • home folder (~) in Win is:
      C:/Users/<your_username>
    • create ~/.bashrc if it does not exist

                # use `alias` to define a shortcut, for example:

                alias c="git checkout"
                __git_complete c _git_checkout

                # always add __git_complete at the end
                # to get git cmd autocomplete functionality, format:
                __git_complete <your_alias> __git_<actual_git_cmd_name>

Commands quick overview

Always remember

  • every git command comes with manual:
    • for short help:$git <cmd> -h
    • for full manual:
      • $git <cmd> --help or
      • $git help <cmd>
  • most git commands come with some sort of dry-run option

Moving around


                $git checkout -b    # branch + checkout
                $git checkout -B    # branch --force + checkout

                $git reset --hard   # for full undo and for jumping around

                $git stash [save]   # quick save work in progress
                $git stash apply    # restore top stashed change
                $git stash drop     # remove from stash last stashed change
                $git stash pop      # stash apply + stash drop

                $git branch -d      # delete branch
                $git branch -D      # force delete branch

Selective adding


                $git gui

                # but also
                $git add -i
                # short for --interactive

                # and as always
                $git commit --verbose --edit
                # to commit with diff overview

Cleaning up


                $git rebase -i <cmd>

                # From there we can:
                # p, pick = use commit
                # r, reword = use commit, but edit the commit message
                # e, edit = use commit, but stop for amending
                # s, squash = use commit, but meld into previous commit
                # f, fixup = like "squash", but discard this commit's log message
                # x, exec = run command (the rest of the line) using shell
                # d, drop = remove commit

                # in case of trouble or fear
                $git rebase --abort
