What git is

About distributed

.git folder

  • repo is folder that contains `.git`
  • working space is everything else
  • .git contains among other things:
    • local config
    • full version history
    • all trackers and flags

git objects

...or how git stores data:

git pointers

are basically anything that resolves to SHA1:
branch, tag, relative pointers with ~ ^ or @{x}

Everything in git stems from understanding of this two pieces of information:

  1. understanding meaning behind `Distributed Version Control System`
  2. how is data being stored (versions)

And that constitutes basic knowledge about git.

Git workspace and workflow

Commands rundown

help ~ remote sync ~ local ~ undoing ~ demo

Always remember

  • every git command comes with manual:
    • for short help:$git <cmd> -h
    • for full manual:
      • $git <cmd> --help or
      • $git help <cmd>
  • for quick-start guide: git help tutorial
    • to list all available guides: $git help -g

Remote sync



                    $git clone `URL`
                    # same as init + remote add + fetch --all + checkout master

                    $git pull <remote>:<branch>
                    # same as fetch <remote>:<branch> + merge <remote>/<branch>

                    $git push [--set-upstream] <remote>
                    # set-upstream only when first time pushing new branch

Local



                    $git status
                    $gitk [--all] [&]
                    # to visualise current state

                    $git branch <branch_name>
                    $git checkout [-b] <branch_tag_or_SHA1>
                    # for moving around

                    $git merge <branch>
                    # for merging in changes from <branch> into current branch

                    $git add [-A | -i]
                    $git commit [-ve | -m "message"]
                    $git gui
                    # for visual and/or partial commiting

Undoing



                    $git merge --abort
                    # when not wanting to deal with conflict resolution

                    $git reset [--hard] HEAD
                    # unstage [throw away] changes
